# 005 — Create Book Page

## Problem Statement

Users have no way to create a new book from the frontend. The API already supports `POST /api/v1/books`, the service layer (`booksService.createBook`) and the store (`useBooksStore.createBook`) are already implemented, but there is no UI to trigger them. Users landing on the empty books list see a message telling them to create a book, but have no actionable button to do so.

## Solution

Add a dedicated "Create Book" page at `/books/new` with a simple form (name + optional currency), plus a "New book" trigger button on the BookPage that navigates to it. After creation the user is redirected back to the books list with a success toast.

## User Stories

1. As a user, I want to see a "New book" button on the books list page, so that I can start creating a book when I need one.
2. As a user on the empty books list, I want to see a "New book" button in the empty state, so that I know where to go to create my first book.
3. As a user, I want to enter a book name and an optional currency, so that my book is configured the way I need it.
4. As a user, I want the currency field to let me type any ISO code even if it's not in the suggested list, so that I can use less common currencies.
5. As a user, I want to see inline validation when I try to submit without a name, so that I know what's missing.
6. As a user, I want to see API errors surfaced on the form, so that I know if something went wrong on the server.
7. As a user, I want a "Cancel" link to go back to the books list without creating anything, so that I can abort if I change my mind.
8. As a user, after successfully creating a book I want to be redirected back to the books list, so that I can see my new book alongside the others.
9. As a user, after creation I want a brief success toast, so that I'm confirmed the action worked.

## Implementation Decisions

### Modules

| Module | Type | Responsibility |
|--------|------|----------------|
| `CurrencyCombobox` | **New shared component** (`components/common/`) | Combobox with a preset list of common currency codes + free-typing support. Accepts `value` and `onChange` props. |
| `CreateBookPage` | **New page** (`pages/books/CreateBookPage.tsx`) | Form with name input + currency combobox. Submits via `useBooksStore.createBook()`. Redirects on success, shows API errors on failure. |
| `BookPage` | **Modified** | Add "New book" button in the empty state and below the book list. Both buttons navigate to `/books/new`. |
| Route config | **Modified** | Add `/books/new` as a child route of `/books`. |

### CurrencyCombobox interface

```tsx
interface CurrencyComboboxProps {
  value?: string
  onChange?: (value: string) => void
  disabled?: boolean
}
```

- Renders a `Combobox` with a list of common ISO currency codes (EUR, USD, GBP, CHF, JPY, CAD, AUD, BRL, CNY, INR, MXN, NOK, SEK, DKK, PLN, CZK, HUF, TRY, ZAR, KRW, SGD, NZD, THB, ILS, SAR, AED).
- The combobox allows free-text typing so the user can enter any ISO code not in the list.
- The typed value is available to the parent via `onChange`.
- Placed in `components/common/` because currency selection is a reusable concept across domains (potential future use in book editing, transaction entry, reporting).

### CreateBookPage flow

1. User navigates to `/books/new`.
2. Renders a centered form card matching the `max-w-2xl mx-auto px-4` layout used by other pages.
3. **Name** field: required text input using the `Field` + `Input` shadcn components.
4. **Currency** field: `CurrencyCombobox`, optional.
5. **Cancel** link (variant "outline" or "ghost") navigates back to `/books`.
6. **Create** button (variant "default") submits the form.
7. On submit: 
   - Client-side validation checks `name` is non-empty. If empty, shows `FieldError` under the name field.
   - Calls `useBooksStore.createBook({ name, currency })`.
   - On success: `navigate("/books")` + `toast.success("Book created")`.
   - On failure: display the API error message as a general form error via `FieldError`.
8. Form state is managed with plain `useState` — no form library.

### BookPage changes

- **Empty state** (the `if (!isLoading && books.length === 0)` branch): add an "outline" `Button` with text "New book" below the paragraph, which calls `navigate("/books/new")`.
- **Books list** (the main return block): add an "outline" `Button` with text "New book" below the card list (after `</div>` closing the `space-y-3` div, before the error block), which calls `navigate("/books/new")`.

### Route changes

```tsx
// Current:
{ path: "books", element: <BookPage /> }

// New:
{
  path: "books",
  children: [
    { index: true, element: <BookPage /> },
    { path: "new", element: <CreateBookPage /> },
  ],
}
```

### What does NOT change

- `API.md` — `POST /api/v1/books` is already documented.
- `handlers.ts` (MSW) — the handler already exists.
- `booksService.ts` — `createBook` already exists.
- `useBooksStore` — `createBook` action already exists.
- Types (`BookDto`, `CreateBookRequest`) — already defined.

## Testing Decisions

### General approach

- Test behavior, not implementation details.
- Use user-visible assertions (queries via `getByRole`, `getByLabelText`, `getByText`, etc.).
- MSW handles network requests — no mocking of the service layer.
- Seed session is `usr_1` (auto-seeded in `test-setup.ts`).

### Modules to test

**CreateBookPage** (`CreateBookPage.test.tsx`):

- Renders the form: name input, currency combobox, Cancel link, Create button.
- Shows validation error when submitting with an empty name.
- Submits successfully and redirects to `/books` + shows success toast.
- Displays API error when the server returns an error.
- Cancelling navigates back to `/books`.

Prior art: `AddPage.test.tsx` tests a form page with MSW-backed submission, toast verification via `waitFor`, navigation checks, and error state rendering.

**BookPage** — no new tests. The button additions are thin UI changes. If an existing test file exists for BookPage, update assertions accordingly; otherwise not required for this change.

**CurrencyCombobox** — no dedicated component test. Tested implicitly through the CreateBookPage test (renders as part of the form, typing a custom value works).

## Out of Scope

- Editing or deleting books from the Create Book page.
- Book sharing or permission management.
- Batch import of books.
- Currency validation against a live exchange rate API.
- Book entity validation beyond the API contract (e.g., duplicate name detection is server-side).

## Further Notes

- The `Combobox` shadcn component is built on `@base-ui/react` and already supports free-typing: the user can type arbitrary text into the input while the dropdown filters matching options. The typed value is always available regardless of whether it matches a preset option.
- Book creation is currently only exposed via API — this PRD closes the gap between "the backend and store support it" and "the UI exposes it".
